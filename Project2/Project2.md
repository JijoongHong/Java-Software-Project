\2) 비교 설명

가. 지향점 및 중점 

- C언어의 경우 순차적으로 자료를 처리하는 절자치향 프로그래밍 방식이다. 이 때 데이터 자체보다는 데이터를 처리하는 로직과 알고리즘을 우선 순위에 둔다. 함수와 변수는 개별적 존재로서 존재하며 함수를 통해 어떻게 변수를 처리하는지 작성한다.
- Java의 경우 알고리즘보다는 객체간의 유기적인 관계가 강조되는 객체지향 프로그래밍이다. 프로그램의 동작 방식보다는 객체를 우선 순위로 둔다. 즉, 사각형이라는 객체에서 함수와 변수는 객체 내에 종속되며, 구성요소의 상태와 동작을 기술하는 방식으로 프로그램을 작성한다. 

나. 캡슐화 

- C언어의 경우 구조체를 통해 여러 데이터를 동시에 관리할 수 있다. 이 때 구조체는 속성, 즉 변수만을 정의할 수 있다. 
- 서로 다른 자료형을 한데 묶어 관리할 수 있다는 점에서는 같지만 Java의 경우 클래스를 선언함으로써 객체의 속성과 더불어 행동을 메소드를 통해 정의할 수 있다. 

다. 생성자

- 구조체를 선언하는데 있어 생성자가 필요없지만, Java의 클래스는 생성자를 필요로 한다. 

라. 클래스 / 구조체 저장위치

- C의 경우 구조체를 메모리의 스택에 저장하여 활용한다.
- Java의 경우 클래스를 힙에 저장하며, 새로운 인스턴스를 생성할 시 클래스를 가리키는 변수를 만들어 스택에 저장하고 이후 힙의 클래스를 가리키도록 한다. 이를 통해 변수가 동일한 클래스를 가리키도록 할 수 있다.

마. 오버로딩

- C언어의 경우 오버로딩이 불가능하다. 
- Java의 경우 클래스 생성에서도 메소드 오버로딩이 가능하다. 아래 사진과 같이 전달받는 인자의 값에 따라 동일한 이름으로 사용이 가능하다. Rectangle 클래스에서도 사각형 생성 시 아무런 인자를 전달하지 않으면 1 x 1 크기의 사각형을 생성하도록 하였다. 

바. 포인터 활용

- C언어의 경우 포인터를 통해 특정 변수의 메모리에 직접 접근할 수 있으며 그 값의 변경이 가능하다. 위 예제에서도 Resize함수에서 call by reference를 통해 직접 구조체의 값에 접근해 값을 변경시켰다. 
- Java의 경우 포인터 사용이 불가능하다. 그러나 객체에 바로 접근함으로써 동일한 결과를 얻을 수 있었다.

사. 가비지 컬렉션

- C언어의 경우 포인터 사용과 같이 메모리에 직접 접근이 가능하다. 마찬가지로 동적할당 기능을 활용하여 개발자의 의도에 따라 수동으로 메모리의 할당과 해제가 이루어진다. 
- Java의 경우 직접적으로 메모리를 다루지 않는다. 그러므로 메모리의 해제를 가비지 컬렉터가 자동으로 수행한다. 

아. 정보은닉

- Java의 경우 private 키워드를 활용해서 외부에서 인스턴스의 변수에 접근하지 못하도록 막을 수 있다. 반대로 이를 공개하려면 public 키워드를 활용한다. 그러나 C언어의 경우 이러한 정보은닉 기능이 기본으로 제공되지 않는다.

자. 실행과 속도

- C언어의 경우 컴파일러가 소스코드를 바로 컴파일 후 OS 위에서 실행한다. 더불어 기계어에 가까운 언어이므로 처리 속도가 빠르다.
- Java의 경우 컴파일러가 바이트코드로 변환하고, 자바가상머신에서 이를 실행한다. 이 경우 C언어에 비해 비교적 속도가 느리다. 

차. 유지보수 

- C언어의 경우 규모가 프로그램의 유지보수가 어렵다. 순차적으로 진행되는 코드 특성 상 일부의 수정이 코드 전체의 수정으로 이어질 수 있으며 디버깅이 어렵다.
- Java의 경우 유지보수를 위해 특정 클래스만 수정하므로 되므로 코드의 수정과 디버깅이 용이하다. 

카. 적합한 프로그램

- 위에서 기술한 C언어의 특성 상 빠른 속도가 요구되고 비교적 소형의 프로그램에 적합하다.
- 위에서 기술한 Java의 특성 상 대형 프로젝트를 다수의 인원이 개발하기에 적합하다.
